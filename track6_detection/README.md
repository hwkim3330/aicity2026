# AI City Challenge 2026 — Track 6: Cross-City Object Detection

Single-image, 10-class vehicle/person detection with geographic
domain-shift (cross-city) evaluation, run on Milestone Systems' **Hafnia
Training-as-a-Service (TaaS)** platform. Participants cannot download the
full dataset or train locally — training happens inside Hafnia on
infrastructure they control, driven by a "trainer package" you upload.

This directory contains everything that could be prepared **without
logging into Hafnia or the AI City evaluation portal**. Items that
genuinely require an account are listed at the bottom under "Remaining
steps (require login)".

## Task summary (confirmed, public)

- **Classes (10, fixed order — index-sensitive for submissions):**
  `Vehicle.Car, Pickup Truck, Single Truck, Combo Truck, Heavy Duty Vehicle,
  Trailer, Motorcycle, Bicycle, Van, Person`
- **Data:** ~13,000 training frames / ~150,000 instances, still images,
  2D axis-aligned boxes. No video/tracking info allowed at inference.
- **Metric:** mAP-family (exact IoU thresholds/averaging not published).
- **Dates:** sample dataset + registration open 2026-05-18; submission
  deadline 2026-07-10 (AoE); workshop at ECCV 2026, Malmö (Sept 8–9).
- Source: https://www.aicitychallenge.org/2026-track6/ and
  https://www.aicitychallenge.org/2026-faqs/ (both public).

## Hafnia platform facts (confirmed, public)

- Python SDK/CLI on PyPI: **`pip install hafnia`** (or `uv add hafnia`),
  currently v0.7.13. https://pypi.org/project/hafnia/
- Public reference repos: `github.com/milestone-hafnia/hafnia` (SDK+CLI+docs),
  `.../trainer-object-detection` (reference trainer, wraps RF-DETR),
  `.../trainer-classification`.
- Docs: https://hafnia.readme.io/docs/welcome-to-hafnia and
  .../getting-started-training-aas (pricing/limits).
- **CLI surface:**
  ```
  hafnia configure | clear
  hafnia profile     ls | active | use | rm | create
  hafnia dataset     ls | download | delete
  hafnia recipe      ls | create | rm
  hafnia trainer     ls | create | update | create-zip | view-zip
  hafnia experiment  ls | create | environments
  hafnia runc        build | build-local | launch-local
  ```
- **Dataset access inside code:** `HafniaDataset.from_name("<name>", version="<v>")`
  — the same call returns the local sample dataset on your laptop and the
  full hidden dataset automatically when run inside a managed job.
- **Trainer package layout (confirmed from the reference repos):**
  ```
  Dockerfile  pyproject.toml  uv.lock  .python-version  .hafniaignore
  scripts/train.py + train.schema.json
  scripts/benchmark.py + benchmark.schema.json
  scripts/export_onnx.py
  src/   tests/
  ```
  No hand-written `config.yaml` — config is CLI args, and
  `*.schema.json` (normally auto-generated by `hafnia trainer create-zip`)
  drives the web dashboard's experiment form.
- **Dockerfile convention:** `debian:bookworm-slim` base + `uv` package
  manager (not a CUDA base image — GPU comes from the platform runtime).
- **Credits/limits — all confirmed against https://hafnia.readme.io/docs/getting-started-training-aas
  and the Track 6 / FAQ pages:**
  - 30,000 credits per participant (Track 6 allocation).
  - Trainer package upload capped at **2GB** (separate 2GB cap on the
    trained model itself).
  - **1 concurrent experiment** max.
  - "Lite" tier: 1× T4 16GB = **83 credits/hr** → ≈361 hrs on the full budget.
  - "Scale" tier: 4× T4 48GB = **546 credits/hr** → ≈55 hrs on the full budget.
  - No external training data allowed; pretrained checkpoints ARE allowed.
  - Ensembles allowed only as a single inference pipeline.
- **Submission is a separate, two-step process:** (1) Hafnia runs
  inference on the hidden benchmark set and produces prediction files;
  (2) you manually download those files and upload them to the official
  AI City evaluation portal, **https://eval.aicitychallenge.org/aicity2026/**
  (own login system, not CodaLab/EvalAI). Portal rules: max 5
  submissions/track/day, 20/track total, org-email registration.

### Model choice: Ultralytics YOLO11 (not Hafnia's RF-DETR reference)

Hafnia's own example trainer wraps **RF-DETR**. We use **YOLO11**
(`scripts/train.py --arch yolo11m.pt`, swappable to `rtdetr-l.pt` for a
transformer receptive field) instead, because:
- Single pip package, extremely stable `train()`/`predict()` API — less
  risk of the trainer package breaking for reasons unrelated to modeling.
- Strong small-object performance (motorcycles, bicycles, distant
  vehicles in traffic-cam footage).
- Native mosaic/mixup/copy-paste augmentation and easy ONNX export.

Trainer packages are "bring your own Docker + Python", so this is
allowed — the entrypoint contract (argparse flags ↔ `*.schema.json`) is
what actually needs to match Hafnia's expectations, not the framework
choice. If it turns out Hafnia's harness strictly requires the RF-DETR
skeleton, adapting `scripts/train.py` to that shape is a light rewrite;
the class-imbalance logic in `src/` is framework-agnostic apart from the
`focal_patch.py` monkeypatch (which is Ultralytics-loss-internals specific).

### Class-imbalance / fine-grained-confusion handling

Traffic-camera footage is expected to be heavily skewed toward
`Vehicle.Car` / `Person`, with `Trailer`, `Combo Truck`,
`Heavy Duty Vehicle`, `Bicycle` scarce — and several classes are visually
confusable (Pickup Truck vs Van vs Car; Single vs Combo Truck vs Heavy
Duty Vehicle vs Trailer; Motorcycle vs Bicycle at range). Practical,
external-data-free mitigations implemented:
1. **`src/class_balance.py`** — image-level oversampling: images
   containing rarer classes are repeated in the training list
   proportional to inverse sqrt class frequency (capped at 8×), computed
   directly from YOLO label files. Dataset-format agnostic, survives
   whatever exact mount/SDK convention Hafnia uses.
2. **`src/focal_patch.py`** — monkeypatches Ultralytics' classification
   BCE loss with `FocalLoss(gamma=1.5)` to concentrate gradient on
   hard/rare/confusable classes instead of the easy majority class.
3. **Copy-paste augmentation** (`--copy-paste 0.3`) — synthesizes extra
   rare-class instances onto other in-dataset backgrounds (no external
   imagery, compliant with the "no external data" rule).
4. **`--close-mosaic 10`** — disables mosaic for the last 10 epochs to
   sharpen box localization right before validation, which matters for
   telling apart visually-similar elongated vehicle classes.
5. Recommended imgsz **960** (not default 640) since motorcycles/
   bicycles/persons are small relative to frame in traffic-cam shots.

### Credit-budget strategy

30,000 credits total, 1 concurrent experiment. Suggested allocation:
1. `--dry-run` locally / via `hafnia runc launch-local` first — free,
   validates the trainer package mechanics before spending any credits.
2. A short Lite-tier (83 cr/hr) shakeout run (~5 epochs, `yolo11m.pt`)
   on the real Hafnia sample dataset to confirm the SDK dataset path,
   loss curves, and schema wiring — a few hundred credits.
3. Spend the bulk of the remaining budget on one or two full runs
   (100 epochs, imgsz 960) on either Lite or Scale tier depending on how
   many GPU-hours the credit math allows once (2) is known precisely.

## Remaining steps (require Hafnia / AI City login)

These cannot be completed from this environment — no credentials here.
In order:

1. **Register** at the Track 6 Hafnia community club page (capped at
   200 accounts, needs an organizational email, ~3–4 day approval):
   https://community.hafnia.milestonesys.com/home/clubs/ai-city-challenge-track-6-omnhs/overview
2. **Create a Hafnia account + API key** at https://hafnia.milestonesys.com/,
   then `pip install hafnia && hafnia configure`.
3. **Confirm the exact Track 6 dataset name** via `hafnia dataset ls` and
   set it as `TRACK6_DATASET_NAME` (env var consumed by
   `trainer_package/scripts/train.py`) — currently a placeholder
   (`REPLACE_ME_TRACK6_DATASET_NAME`).
4. `hafnia dataset download <track6-name>` to get the local sample
   dataset for real pipeline testing (replaces the toy-data fallback in
   `data.yaml`).
5. Run `uv lock` inside `trainer_package/` with `hafnia` resolvable, so
   the Dockerfile's `uv sync --locked` has a real lockfile.
6. `hafnia trainer create-zip trainer_package/` then
   `hafnia runc build-local` + `hafnia runc launch-local` to sanity-check
   the container locally before spending credits — also regenerates the
   real `train.schema.json`/`benchmark.schema.json` to diff against the
   hand-written placeholders here.
7. `hafnia experiment create` to launch the actual (metered) training
   job on Hafnia's infrastructure.
8. Run benchmarking on the platform, download the resulting prediction
   file(s), and inspect their real schema — adapt
   `scripts/benchmark.py`'s `write_submission()` / `write_hafnia_predictions()`
   to match exactly.
9. Register at the AI City evaluation portal
   (https://eval.aicitychallenge.org/aicity2026/, separate org-email
   registration, 24–48hr approval) and submit the prediction file
   (max 5/day, 20 total for Track 6).
